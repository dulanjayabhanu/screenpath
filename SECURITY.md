# Security Policy

## Supported Versions

Security updates are applied to the most recent version of the project.
Older versions may not receive security fixes.

| Version        | Supported |
| -------------- | --------- |
| Latest         | Yes       |
| Older versions | No        |

<br>

## Reporting a Vulnerability

If you discover a security vulnerability in ScreenPath, please report it responsibly.

Do not open a public issue for security vulnerabilities.

Instead, report the issue privately so it can be investigated and resolved before public disclosure.

You may report vulnerabilities through one of the following methods:

* Open a private security advisory through GitHub
* Contact the maintainer directly via email

Please include the following information in your report:

* A clear description of the vulnerability
* Steps required to reproduce the issue
* Potential impact of the vulnerability
* Any suggested mitigation or fix (if available)

<br>

## Response Process

After receiving a vulnerability report:

1. The issue will be reviewed and validated.
2. A fix will be developed if the vulnerability is confirmed.
3. The fix will be released in the next security update.
4. The vulnerability may be publicly disclosed once it has been resolved.

<br>

## Responsible Disclosure

We request that security researchers and contributors follow responsible disclosure practices.

Please allow sufficient time for the vulnerability to be investigated and patched before publicly sharing details.

This approach helps protect users and ensures the issue can be resolved safely.

<br>

## Scope

The security policy applies to the entire repository, including:

* application source code
* configuration files
* dependency usage
* deployment configurations

<br>

Thank you for helping keep ScreenPath secure.
